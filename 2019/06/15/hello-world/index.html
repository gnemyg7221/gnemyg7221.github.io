ConstraintLayout

一. 布局

1. 居中

居中于⽗父容器

居中于控件中⼼心

  • ⽔平⽅向居中

  • 垂直⽅向居中

居中于控件的边
控件垂直居中于 view 的「下边」

2. 填充

⽔平⽅向填充⽗容器(通过 match_constraint )

备注:在早期版本中 match_parent 没有效果，必须使⽤ match_constraint 来完成。

3. 权重

为⽔平⽅向的控件设置权重，⼤小为 2:1:1 。

4. 文字基准线对⻬

5.圆形定位

通过「圆⼼」「⻆度」「半径」设置圆形定位

二. 特殊属性

1. 约束限制

限制控件⼤大⼩小不不会超过约束范围。

2. 偏向

控制控件在垂直⽅向的 30%的位置

除了了配合百分⽐定位，还有⽤于配合有时在「约束限制」的条件下不不需要居中效果的情况.

垂直⽅向居顶部

3. 约束链

在约束链上的第⼀个控件上加 chainStyle ，⽤来改变⼀组控件的布局⽅式

  • packed(打包)
  • spread (扩散)
  • spread_inside(内部扩散)

垂直⽅向 packed

4. 链宽⾼比

  • 至少需要⼀个⽅向的值为 match_constraint
  • 默认的都是宽高⽐，然后根据另外⼀条边和⽐例例算出 match_constraint 的值

x:y 默认表示的都是 width:height

有值的边，ratio 值(默认是宽高⽐)，被约束的边(也有可能是有值的边)。

  • 宽是 0dp，高是 100dp，ratio 是 2:1
    默认情况下是宽是 200dp，但是我们可以指定被约束的边是 height，那么宽度就是50 dp

  • 高是 0dp，宽是 100 dp，ratio 是 2:1
    默认情况下是高是 50 dp，但是我们指定被约束的边是 width，那么高度为200dp

只有⼀个方向match_constraint不要去⽤，如果是两个方向都是match_constraint 那么可能会⽤到。

5. 百分⽐布局

  • 需要对应⽅向上的值为match_constraint
  • 百分比是parent父容器 的百分比，⽽不是约束区域的百分⽐

宽度是⽗容器的 30%

三. 辅助控件

1. GuideLine

  • 设置辅助线的⽅向 android:orientation="vertical"
  • 设置辅助线的位置，根据⽅向不不同
    • 距离左侧或上侧的距离 layout_constraintGuide_begin
    • 距离右侧或下侧的距离 layout_constraintGuide_end
    • 百分⽐ layout_constraintGuide_percent

2. Group

通过 constraint_referenced_ids 使⽤引⽤的⽅式来避免布局嵌套。 可以为一组控件统一设置 setVisibility

3.Layer

和 Group 类似，同样通过引⽤的⽅式来避免布局嵌套，可以为一组控件统一设置旋转/缩放/位移。

4. Barrier

通过设置一组控件的某个⽅向的屏障，来避免布局嵌套。

5.Placeholder

通过 setContentId 来将指定控件放到占位符的位置。

6.Flow

通过引⽤的⽅式来避免布局嵌套。
wrapMode

  • chain
  • aligned
  • none(默认)

注意这个控件是可以被测量量的，所以对应方向上的值可能需要被确定(即不能只约束同一⽅向的单个约束)

7.ConstraintSet

使⽤ ConstraintSet 对象来动态修改布局。

防⽌止布局中有⽆ id 控件，可以设置 isForceId = false

通过 ConstraintSet#clone 来从 xml 布局中获取约束集。

8.布局扁平化更加容易做过渡动画

在布局修改之前 通过加上 TransitionManager 来⾃动完成过渡动画。
